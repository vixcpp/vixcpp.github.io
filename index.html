What is Vix?

Vix.cpp is not just another backend framework it is a modular C++ runtime designed for distributed applications, edge systems, offline devices and unstable, low-quality networks.

Built for unstable networks

Most runtimes assume stable connectivity. Vix.cpp is engineered to keep working when latency spikes, packets drop and connectivity is intermittent.

Offline-first by design

The goal is to let applications continue running and providing value even when the cloud is not reachable sync when you can, work when you can’t.

Peer-to-peer ready

Vix.cpp is evolving towards a P2P-friendly model where nodes can communicate, sync state and coordinate locally without a central server.

Modern C++ runtime

Rebuilt from scratch in C++20, inspired by FastAPI, Vue.js, React and modern runtimes like Node, Deno and Bun but with native speed and full control.

Why Vix exists

Traditional cloud frameworks assume stable networks, predictable latency and always-online connectivity. In most of the world, this is simply not true.

Offline-first reality

Many users work in environments where connectivity is unreliable. Vix.cpp aims to keep applications usable, even when the network disappears.

Edge & local-first

Move computation closer to the user: edge devices, local clusters, and hybrid systems where cloud is only one part of the picture.

Key features

Async HTTP server

A modern HTTP stack built on top of Beast/Asio, with fast routing and predictable threading.

WebSocket engine

WebSocket module with rooms, metrics and integration with the runtime executor for real-time systems.

ORM for MySQL/SQLite

Database layer for relational workloads, designed to integrate with Vix.cpp’s runtime model.

Middleware system

Structured middleware pipeline for auth, logging, metrics and cross-cutting concerns.

Developer-friendly CLI

vix new, vix build, vix run and script mode to simplify the CMake/toolchain story for C++ developers.

Modular design

Core, utils, websocket, ORM, JSON, CLI, Rix library and more — each module focused and composable.

Real-time WebSocket chat, built with Vix.cpp

Quick example

A minimal HTTP server with a JSON response, using App and a single route.

C++ · Minimal HTTP app 
#include <vix.hpp>
    using namespace Vix;

    int main() {
        App app;

        app.get("/", [](auto&, auto& res) {
            res.json({ "message", "Hello world" });
        });

        app.run(8080);
    }

Script mode — run .cpp files directly

Vix.cpp can execute a single .cpp file like a script — no manual CMake setup, no project boilerplate.

Shell · Run C++ file
vix run file.cpp

Under the hood, Vix generates a temporary CMake project in .vix-scripts/<filename>/, builds it and runs the resulting executable.

Shell · Script mode output
~/myapp/test$ vix run server.cpp
    Script mode: compiling server.cpp
    Using script build directory:
    • .vix-scripts/server

    ✔ Build succeeded
    [I] Server running on port 8080
    ^C
    ℹ Server interrupted by user (SIGINT)

Get started

Clone the repository and try the built-in examples:

Shell · Build Vix.cpp
git clone https://github.com/vixcpp/vix.git
    cd vix
    cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
    cmake --build build -j
    ./build/hello_routes

Once Vix.cpp is installed, you can generate a new project using the CLI:

Shell · Create & run project
vix new myapp
    cd myapp
    vix build
    vix run

For more examples, architecture details and module documentation, check the README and the documentation inside the repository.
